# GitHub Profile README Design

## Goal

Create an impact-first GitHub profile README for Sumeet Das. The profile should feel professional and recruiter-friendly while retaining a little developer personality.

## Positioning

Present Sumeet as a Senior Android and Kotlin Multiplatform developer who builds scalable mobile products and improves delivery through reusable architecture.

The opening should emphasize Kotlin, Jetpack Compose, Compose Multiplatform, and Android architecture. Supporting proof should include:

- 4.8+ years of Android development experience.
- A KMM-to-Compose-Multiplatform migration with 70%+ shared code across Android and iOS.
- A shared multi-module SDK used across 13+ Android applications.
- 100K+ downloads for the flagship education application.
- A 70% reduction in development time across related applications.

## Structure

1. Centered name and concise professional headline.
2. Short introduction focused on the value Sumeet delivers.
3. About section containing measurable career highlights.
4. Focused technology stack using readable badges.
5. Selected-work section covering the crypto exchange migration, Vantero, enterprise POS work, and the education-app SDK.
6. Current interests covering Compose Multiplatform, architecture, performance, and agentic development workflows.
7. Contact section with email, LinkedIn, Stack Overflow, portfolio, and resume links.
8. Compact GitHub statistics cards as supporting material rather than the main content.

## Content Rules

- Prefer measurable outcomes over generic claims.
- Keep the profile concise enough to scan in approximately one minute.
- Use current Android/Kotlin technologies and omit outdated or weakly relevant tools.
- Correct the malformed LinkedIn link from the existing README.
- Preserve the supplied Google Drive resume URL unless a newer public URL is provided.
- Avoid excessive animations, decorative widgets, visitor counters, and long icon walls.
- Use GitHub-compatible Markdown with limited HTML only where alignment materially improves presentation.

## Deliverable

Create a standalone Markdown file containing copy-ready content for the GitHub profile repository README. Do not overwrite the portfolio website README or unrelated project files.

## Acceptance Criteria

- The headline clearly identifies Sumeet's Android and Kotlin Multiplatform specialization.
- At least three verified, measurable outcomes appear above the fold or in the first major section.
- The skills list reflects the technologies evidenced by the portfolio.
- All supplied contact links are syntactically valid.
- The result renders correctly as a GitHub profile README.
- No placeholders or invented links remain in the final file.
